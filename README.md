# DAVIAN LaTeX Template

A clean, single-column LaTeX template for arXiv-style preprints and technical
reports, branded for **DAVIAN Lab, KAIST AI**. Styled after modern AI tech
reports (XCharter body, Montserrat headings, Charter-matched math, KAIST-blue
accent).

## Quick start

```bash
latexmk -pdf          # builds build/main.pdf
latexmk -C            # clean all generated files
```

Requires a full TeX Live (2025 tested) with `xcharter`, `newtx`, `montserrat`,
`inconsolata`, `titlesec`, `natbib`, `cleveref`, `microtype`. Compiles with
**pdfLaTeX**.

## Repository layout

```
davian.cls            # the document class (fonts, colors, headings, title block)
latexmkrc             # build config — output goes to build/
references.bib        # bibliography (BibTeX, used with plainnat)
main.tex              # thin orchestrator: metadata + \input of preambles/sections

preambles/
  libs.tex            # document-level \usepackage lines + \graphicspath
  macros.tex          # paper-specific shorthand macros
  fig_comm.tex        # figure definitions — one \newcommand per figure
  tbl_comm.tex        # table definitions  — one \newcommand per table

sections/
  00_abstract.tex     # abstract + \keywords
  01_introduction.tex
  02_method.tex
  03_experiments.tex
  04_conclusion.tex

figs/                 # paper figures (PDF/PNG) — \includegraphics{figs/...}
tbls/                 # table images/PDFs    — \includegraphics{tbls/...}
assets/               # brand assets: KAIST/DAVIAN logos + generator scripts
```

This modular layout mirrors a real paper workflow: `main.tex` stays short,
each section lives in its own file, and floats are defined once in
`preambles/fig_comm.tex` / `tbl_comm.tex` then *called by name* where they
should be anchored.

## Writing your paper

**1. Metadata** — edit the title block at the top of `main.tex`:

```latex
\shortauthor{Last Name et al.}                 % left side of running header
\runningtitle{Short title for the page header} % right side of running header
\title{Your Paper Title}

\authors{%
  First Author\affmark{1,*}\quad
  Second Author\affmark{1,\dag}%
}
\affiliations{%
  \affmark{1}DAVIAN Lab, KAIST AI \qquad
  \affmark{2}Other Institution%
}
\contributions{%
  \affmark{*}Equal contribution.\quad
  \affmark{\dag}Corresponding author.%
}
\correspondence{\email{you@kaist.ac.kr}}   % optional; \email = mono + mailto link
```

- `\affmark{...}` renders a superscript marker — use it both on author names
  and in the affiliation list to link them.
- `\contributions` and `\correspondence` are optional; omit them if unused.
- `\email{addr}` typesets the address in monospace and hyperlinks it via `mailto:`.
- Defaults point to DAVIAN Lab, KAIST AI, so the template is usable as-is.

**2. Body** — write each section in its own `sections/NN_*.tex` file and
`\input` it from `main.tex`. Use `\section`, `\subsection`, `\paragraph`,
`equation`, and `\cite{...}`; cross-reference with `cleveref`'s `\Cref{...}`.

**3. Figures & tables** — define each as a `\newcommand` in
`preambles/fig_comm.tex` / `tbl_comm.tex`, then call it in the section body
(e.g. `\figPlaceholder`, `\tblResults`). Drop image files in `figs/` / `tbls/`;
`\graphicspath` resolves them so no path prefix is needed in `\includegraphics`.

## Header logos

The first page shows the KAIST AI logo (left) and DAVIAN logo (right) in the
header, with a rule beneath. The class looks for these files and falls back to a
typeset wordmark if absent:

```
assets/kaist-ai-logo.pdf   # preferred (vector); .png is the fallback
assets/davian-logo.pdf     # preferred (vector); .png is the fallback
```

The committed PNGs are generated by the scripts in `assets/`
(`render_kaist_logo.py`, `make_davian_text_logo.py`). To use a different logo,
override in your preamble, e.g.:

```latex
\renewcommand{\davianlogo}{\includegraphics[height=0.35in]{assets/mylogo.pdf}}
```

## Customization

All styling lives in `davian.cls`:

- **Accent color** — edit the `kaistblue` / `kaistbluedark` definitions.
- **Abstract box shade** — edit the `abstractbg` color.
- **Heading font** — swap the `montserrat` package line for `FiraSans`.
- **Title-block spacing** — the `\vskip` values inside `\maketitle`.
- **Margins / header height** — adjust the `geometry` options.
- **Header/footer rules** — edit the `davianmain` / `davianfirst` page styles.
